# RentRadar — Мониторинг аренды студий СПб

## 1. Проблема

Арендодатель с несколькими квартирами (студии, район Подольской ул.) не знает реальную рыночную цену аренды в моменте. Ручной мониторинг Авито + ЦИАН занимает 2-3 часа в неделю. Без истории цен невозможно понять тренд: рынок растёт или падает. Без данных о конкурентах — невозможно оптимально выставить цену.

## 2. Решение

Автоматическая система:
1. **Парсер** — ежедневно собирает объявления об аренде студий с Авито и ЦИАН в заданной зоне
2. **Хранилище** — SQLite база с историей всех объявлений и ценовых snapshot-ов
3. **Аналитика** — расчёт мин/макс/средней цены, количества, длительности размещения
4. **Дашборд** — HTML-сайт на VPS с графиками, таблицей, фильтрами

## 3. Целевая аудитория

Единственный пользователь — владелец (Geser). Дашборд открыт по прямой ссылке, без авторизации.

## 4. Архитектура

│  (httpx + API)  │    │  (Playwright)   │
└────────┬────────┘    └────────┬────────┘
         │                      │
         ▼                      ▼
    ┌──────────────────────────────┐
    │       SQLite Database        │
    │  listings / price_history /  │

                   │

    ┌──────────────────────────────┐
    │   JSON Generator (Python)    │
    │  → /var/www/rentradar/data/  │
    └──────────────┬───────────────┘
                   │
                   ▼
    ┌──────────────────────────────┐

### Стек
- **Python 3.11+** — парсер, аналитика, генерация JSON
- **Playwright** — headless browser для Авито
- **httpx** — HTTP-клиент для ЦИАН API
- **SQLite** — хранилище (файл `rentradar.db`)
- **APScheduler** — планировщик задач (или systemd timer)
- **nginx** — статика для дашборда
- **Chart.js 4** — графики
- **HTML/CSS/JS** — дашборд (vanilla, без фреймворков)

## 5. Параметры парсинга

### Зона
Границы: Фонтанка (север) → Рузовская ул. (восток) → Обводный канал (юг) → Измайловский пр-т (запад)

Улицы внутри зоны:
Подольская, Серпуховская, Верейская, Бронницкая, 1-5-я Красноармейские, Батайский пер., Дойников пер., Клинский пр., Малодетскосельский пр., Можайская, Рузовская

### Фильтры
- Тип: студия
- Площадь: 0–35 м²
- Аренда: 5 000 – 65 000 ₽
- Сдача: все (фиксируем тип: собственник / агент / с комиссией)
- Коммуналка: парсим отдельным полем
- Счётчики: парсим если указано в объявлении

### Частота
- Парсинг: 1 раз в день (06:00 MSK)
- Telegram-алерты: нет

## 6. Дашборд — структура блоков

### Блок 1 — Сводка (верх)
4 карточки в ряд: мин. цена / макс. цена / средняя цена / кол-во объявлений.
Каждая с дельтой за неделю (↑↓).

### Блок 2 — График цен
Линейный график (Chart.js): три линии (мин/средн/макс).
- Переключатель периода: 30д / 60д / 90д
- Переключатель шага: день / неделя / месяц
- При группировке по неделям — диапазоны дат, точки на линиях

### Блок 3 — Таблица объявлений
Сортируемая таблица. Колонки:
- Адрес
- Площадь (м²)
- Аренда (₽)
- Коммуналка (₽)
- Кто сдаёт (бейдж: собственник / агент)
- Площадка (бейдж: ЦИАН / АВИТО)
- Возраст объявления (бейдж: 🟢 0-7д / 🟡 8-15д / 🔴 21+д)
- Ссылка (активная, target=_blank на оригинал)

### Блок 4 — Аналитика предложения
Столбчатый stacked график: Авито + ЦИАН.
Переключатель шага: день / неделя / месяц.

### Футер
«Последнее обновление: [дата] · парсер v1.0 · разработано Gesolutions©»

## 9. Риски

| Риск | Вероятность | Митигация |
|---|---|---|
| Авито блокирует парсер | Высокая | Playwright + ротация User-Agent + задержки 3-7с |
| ЦИАН меняет API | Средняя | Мониторинг ответов, fallback на Playwright |
| VPS не хватает RAM для Playwright | Низкая | 2GB достаточно, swap 1GB |
| Дублирование объявлений | Средняя | Дедупликация по URL + адрес + цена |
| Объявления вне зоны в результатах | Средняя | Фильтрация по списку улиц после парсинга |
